0x60 图论

Posted on 2022-11-03 Edited on 2022-11-06
图论问题的最基本的算法模板,综合应用和扩展应用博主一般是写题解，实在没时间写博客(太菜了qaq)

单源最短路

dijkstra O（n^2）

不用多说，单元最短路的第一关，利用贪心的思想，不断选择距离原点最近的点加入集合，用加入集合的该点更新所有集合外的点到原点的距离(将起点加入集合后只需n-1次迭代就可得到所有点最短路)

可处理有向无向有环无环

堆优化（mlogn）

既然dijkstra每轮找离原点最远的且没加入集合的点进入集合，我们很容易想到用一个小根堆去优化它

记得把距离放在pair的first位，堆依据这个（距离）排序，每个点一旦出堆就一定是最小值

两种算法不能说谁一定比谁好，例如在处理稠密图的时候堆优化效率不如朴素版dijkstra，因为看代码可知，每更新一个点（一条边），就将更新的点插入堆中，那么当一个点被多次更新的时候，堆中会堆积多个点（但堆是按距离排好序，只会对最小距离的点做松弛操作，所以保证了正确性），但对于稠密图（很多边）频繁的push，pop操作会使算法性能下降，具体使用哪个算法请依据实际情况而定

前面说过，dijkstra基于贪心思想，注定目光短浅，也是它不能处理一些带负权边的图

如图，尝试分析

因此我们需要新的算法处理带负权边图

bellman-ford O(nm)

利用dp思想，这个算法类似于从起点开始扩散，i循环控制扩散的边数限制，每次i循环一次都会遍历所有边，利用扩散和三角不等式更新所有边，适合求有边数限制的最短路(改i循环的次数)，最多n-1次迭代会得到所有点的最小值，可以求带负权边的最短路

spfa O(nlogn)

我们发现，在belmanford算法种i每循环一次，都会遍历更新所有边，然而有些时候大多数边不需要更新，很浪费时间，一个点要被更新那么它前面的点一定会被更新，于是我们可以用一个queue存储每一轮被更新的点，只用这些被更新的点去更新其他点，用队列优化bellmanford，即spfa

spfa同样基于dp思想，由于它是bf算法的优化，所以我们一般常用spfa

多源最短路

floyd O(n ^ 3)

多源最短路通用算法，利用dp思想

同时，我们假定k就是j，我们发现k的状态与k-1是一致的，那么我们可以消掉第一维k

同时floyd算法还可用于判断矛盾，传递闭包等问题

最小生成树

prim O(n ^ 2)

思路与dijkstra极其相似，唯一不同是选择距离集合最短的点加入集合而非距离远点最短的点

同样的思路，我们可以尝试堆优化

分析与dijklstra完全相同，不再赘述

kruskal O(mlogm)

完全分析边，按照边从小到大排序，用并查集维护每个连通块，贪心加边，只要两点不在同一个连通块中就连上这条边,连完后一定是n个点n-1条边， 否则无最小生成树

个人认为后者在处理最小生成树问题时一般优于前者，prim能方便处理的问题kruskal一般能很方便处理，反之却不一定

负环

spfa（我们一般使用spfa判断负环，效率更高）

有两种方法

1.开一个cnt数组记录每一个点被更新的次数，如果任意一个点被更新超过n次，可断定存在负环

2.开一个cnt数组用来记录最短路有多少条边，如果超过n条可判断存在负环

很明显，第二种方法更优：

证明其正确性可用反证法，如果某一最短路径等于n条边，那么如果不存在环的话就一定会有n+1个点，但我们只有n个点，根据抽屉原理，必然有点重复，即必然存在环，与假设矛盾

再谈谈为什么第二种方法优于第一种方法

如图

如果是单纯判断负环的话，可以将spfa的队列换成手写栈，防止tle，但是求最短路的时候千万不要把队列换栈，效率极低

原因：spfa每个点出一次队列就更新一次，并作为新的点去更新其他点，每个点可能多次入队出队，于是当前面的点陆续出队时，后面的点也不断在被更新，等到后面的点出队时已经处于较新的状态，而栈每更新一次就出一次，导致大量点被积压在栈内无法弹出，容易堆积，所以用栈判断负环更快

差分约束

本质是一种思路，用于求大量不等式组的可行解，例如

x1 <= x2 + c

x2 <= x3 + c

.

.

.

xn-1 <= xn + c

xn <= 0

对于不等式xi <= xj + c，我们发现这是满足单源最短路的状态计算的，于是我们可以利用单源最短路思想建图，对例子，我们可以建立一条从j到i边权为c的边，同样对于正确性我们可以用反证法，如果d[i] < d[j] + c，那i一定会被更新，同样，对于xi >= xj + c,我们就建一条从j到i边权为c的边，不过我们求的就是最长路了

注意：存在环可能无解，具体情况具体分析（可以用判断负环正环spfa，如果有解顺便还可以求最短路或者最长路，改个符号）

如果我们要求最大值或最小值(一定要有绝对关系，否则全是相对关系，不存在最大最小值)：

最大值求最短路：

对一个点xi <= xi-1 + c1 <= xi-2 + c1 + c2 <= … <= x0 + c1+ … + cn （x0为0，c1 + … + cn即为路径长度），对所有x <= n1, x <= n2 …取交集，即找最小的那个限制条件，及最短路

同理最小值求最长路

次小生成树

定理：如果一个无向图存在最小生成树和（严格）次小生成树，那么对于它的任意一个最小生成树都有一个（严格）次小生成树与之对应

求严格次小生成树步骤：先按kruskal算法思路求得最小生成树，然后遍历未被选中(不在最小生成树内)的边，若当前遍历得边两端点为a,b，我们就连上a,b，设连上ab的边为e，这时就会形成一个环，我们在这个环中选取一条最大的边（除e外，且该边是<=e的），枚举所有点对，取同样操作下所有点对的最小值即为次小生成树。难点就是如何求出树中任意两点间的最大边长度和次大边长度(假如最大边刚好等于e，那就不是严格次小生成树)

我们可以用LCA中的跳跃倍增法求区间最长段
